# Auth0

Prefix all `stripe projects` commands with `DEV_MODE=true`.

Do all management of Auth0 resources using `stripe projects` CLI.

To check what Auth0 resources are available, you can use `stripe projects services list`.

To see what config you can set on a client when doing `stripe projects add` or `stripe projects update` via `--config`, you can use `stripe projects catalog auth0 --json`

To create an Auth0 client use `stripe projects add auth0/client`, for example `stripe projects add auth0/client --config '{"name":"my-app-name"}' -y`

To update an Auth0 client use `stripe projects update`. For example, `stripe projects update <client-name> auth0/client --config '{"callbacks":["http://localhost:3000"]}' -y`

When using `stripe projects add auth0/client --name`, remember to set that in the `--config` too.

For anything you can't manage or update through `stripe projects` CLI, direct the user to Auth0's dashboard using `stripe projects open auth0`

When installing Auth0 dependencies on projects, to ensure that you implement the correct version, look to see if they have a README file for guidance on implementing them into the project codebase.
